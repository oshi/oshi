1   /**
2    * Common OSHI API interfaces, abstract base classes, annotations, and utilities shared by all OSHI implementations.
3    * <p>
4    * This module contains <i>no native code</i> and is unaffected by <a href="https://openjdk.org/jeps/472">JEP 472</a>
5    * restrictions on native access. It can be used standalone by applications that cannot or prefer not to enable native
6    * access (e.g., in restricted JVM environments).
7    * <p>
8    * To build a native-free OSHI implementation:
9    * <ol>
10   * <li>Depend on this module only (Maven: {@code com.github.oshi:oshi-common}).</li>
11   * <li>Extend the abstract base classes in {@code oshi.hardware.common} and {@code oshi.software.common}. For hardware,
12   * subclass {@code AbstractHardwareAbstractionLayer} and override its {@code protected abstract create*()} factory
13   * methods to return your platform-specific implementations. For software and individual components (e.g.,
14   * {@code AbstractOperatingSystem}, {@code AbstractCentralProcessor}, {@code AbstractSensors}), override the
15   * {@code protected abstract query*()} methods with platform-specific logic using command-line tools
16   * ({@code oshi.util.ExecutingCommand}), {@code /proc} file parsing ({@code oshi.util.FileUtil},
17   * {@code oshi.util.ProcUtil}), or other non-native techniques.</li>
18   * </ol>
19   */
