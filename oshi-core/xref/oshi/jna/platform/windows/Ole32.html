143     /**
144      * Registers security and sets the default security values for the process.
145      *
146      * @param pSecDesc
147      *            [in, optional] The access permissions that a server will use
148      *            to receive calls. This parameter is used by COM only when a
149      *            server calls CoInitializeSecurity. Its value is a pointer to
150      *            one of three types: an AppID, an IAccessControl object, or a
151      *            SECURITY_DESCRIPTOR, in absolute format. See the Remarks
152      *            section for more information.
153      * @param cAuthSvc
154      *            [in] The count of entries in the asAuthSvc parameter. This
155      *            parameter is used by COM only when a server calls
156      *            CoInitializeSecurity. If this parameter is 0, no
157      *            authentication services will be registered and the server
158      *            cannot receive secure calls. A value of -1 tells COM to choose
159      *            which authentication services to register, and if this is the
160      *            case, the asAuthSvc parameter must be NULL. However, Schannel
161      *            will never be chosen as an authentication service by the
162      *            server if this parameter is -1.
163      * @param asAuthSvc
164      *            [in, optional] An array of authentication services that a
165      *            server is willing to use to receive a call. This parameter is
166      *            used by COM only when a server calls CoInitializeSecurity. For
167      *            more information, see SOLE_AUTHENTICATION_SERVICE.
168      * @param pReserved1
169      *            [in, optional] This parameter is reserved and must be NULL.
170      * @param dwAuthnLevel
171      *            [in] The default authentication level for the process. Both
172      *            servers and clients use this parameter when they call
173      *            CoInitializeSecurity. COM will fail calls that arrive with a
174      *            lower authentication level. By default, all proxies will use
175      *            at least this authentication level. This value should contain
176      *            one of the authentication level constants. By default, all
177      *            calls to IUnknown are made at this level.
178      * @param dwImpLevel
179      *            [in] The default impersonation level for proxies. The value of
180      *            this parameter is used only when the process is a client. It
181      *            should be a value from the impersonation level constants,
182      *            except for RPC_C_IMP_LEVEL_DEFAULT, which is not for use with
183      *            CoInitializeSecurity. Outgoing calls from the client always
184      *            use the impersonation level as specified. (It is not
185      *            negotiated.) Incoming calls to the client can be at any
186      *            impersonation level. By default, all IUnknown calls are made
187      *            with this impersonation level, so even security-aware
188      *            applications should set this level carefully. To determine
189      *            which impersonation levels each authentication service
190      *            supports, see the description of the authentication services
191      *            in COM and Security Packages. For more information about
192      *            impersonation levels, see Impersonation.
193      * @param pAuthList
194      *            [in, optional] A pointer to SOLE_AUTHENTICATION_LIST, which is
195      *            an array of SOLE_AUTHENTICATION_INFO structures. This list
196      *            indicates the information for each authentication service that
197      *            a client can use to call a server. This parameter is used by
198      *            COM only when a client calls CoInitializeSecurity.
199      * @param dwCapabilities
200      *            [in] Additional capabilities of the client or server,
201      *            specified by setting one or more
202      *            EOLE_AUTHENTICATION_CAPABILITIES values. Some of these value
203      *            cannot be used simultaneously, and some cannot be set when
204      *            particular authentication services are being used.
205      * @param pReserved3
206      *            [in, optional] This parameter is reserved and must be NULL.
207      * @return This function can return the standard return value E_INVALIDARG,
208      *         as well as the following values.
209      *
210      *         S_OK Indicates success.
211      *
212      *         RPC_E_TOO_LATE CoInitializeSecurity has already been called.
213      *
214      *         RPC_E_NO_GOOD_SECURITY_PACKAGES The asAuthSvc parameter was not
215      *         NULL, and none of the authentication services in the list could
216      *         be registered. Check the results saved in asAuthSvc for
217      *         authentication serviceâ€“specific error codes.
218      *
219      *         E_OUT_OF_MEMORY Out of memory.
220      */

224     /**
225      * Sets the authentication information that will be used to make calls on
226      * the specified proxy. This is a helper function for
227      * IClientSecurity::SetBlanket.
228      *
229      * @param pProxy
230      *            [in] The proxy to be set.
231      * @param dwAuthnSvc
232      *            [in] The authentication service to be used. For a list of
233      *            possible values, see Authentication Service Constants. Use
234      *            RPC_C_AUTHN_NONE if no authentication is required. If
235      *            RPC_C_AUTHN_DEFAULT is specified, DCOM will pick an
236      *            authentication service following its normal security blanket
237      *            negotiation algorithm.
238      * @param dwAuthzSvc
239      *            [in] The authorization service to be used. For a list of
240      *            possible values, see Authorization Constants. If
241      *            RPC_C_AUTHZ_DEFAULT is specified, DCOM will pick an
242      *            authorization service following its normal security blanket
243      *            negotiation algorithm. RPC_C_AUTHZ_NONE should be used as the
244      *            authorization service if NTLMSSP, Kerberos, or Schannel is
245      *            used as the authentication service.
246      * @param pServerPrincName
247      *            [in, optional] The server principal name to be used with the
248      *            authentication service. If COLE_DEFAULT_PRINCIPAL is
249      *            specified, DCOM will pick a principal name using its security
250      *            blanket negotiation algorithm. If Kerberos is used as the
251      *            authentication service, this value must not be NULL. It must
252      *            be the correct principal name of the server or the call will
253      *            fail. If Schannel is used as the authentication service, this
254      *            value must be one of the msstd or fullsic forms described in
255      *            Principal Names, or NULL if you do not want mutual
256      *            authentication. Generally, specifying NULL will not reset the
257      *            server principal name on the proxy; rather, the previous
258      *            setting will be retained. You must be careful when using NULL
259      *            as pServerPrincName when selecting a different authentication
260      *            service for the proxy, because there is no guarantee that the
261      *            previously set principal name would be valid for the newly
262      *            selected authentication service.
263      * @param dwAuthnLevel
264      *            [in] The authentication level to be used. For a list of
265      *            possible values, see Authentication Level Constants. If
266      *            RPC_C_AUTHN_LEVEL_DEFAULT is specified, DCOM will pick an
267      *            authentication level following its normal security blanket
268      *            negotiation algorithm. If this value is none, the
269      *            authentication service must also be none.
270      * @param dwImpLevel
271      *            [in] The impersonation level to be used. For a list of
272      *            possible values, see Impersonation Level Constants. If
273      *            RPC_C_IMP_LEVEL_DEFAULT is specified, DCOM will pick an
274      *            impersonation level following its normal security blanket
275      *            negotiation algorithm. If NTLMSSP is the authentication
276      *            service, this value must be RPC_C_IMP_LEVEL_IMPERSONATE or
277      *            RPC_C_IMP_LEVEL_IDENTIFY. NTLMSSP also supports delegate-level
278      *            impersonation (RPC_C_IMP_LEVEL_DELEGATE) on the same computer.
279      *            If Schannel is the authentication service, this parameter must
280      *            be RPC_C_IMP_LEVEL_IMPERSONATE.
281      * @param pAuthInfo
282      *            [in, optional] A pointer to an RPC_AUTH_IDENTITY_HANDLE value
283      *            that establishes the identity of the client. The format of the
284      *            structure referred to by the handle depends on the provider of
285      *            the authentication service. For calls on the same computer,
286      *            RPC logs on the user with the supplied credentials and uses
287      *            the resulting token for the method call. For NTLMSSP or
288      *            Kerberos, the structure is a SEC_WINNT_AUTH_IDENTITY or
289      *            SEC_WINNT_AUTH_IDENTITY_EX structure. The client can discard
290      *            pAuthInfo after calling the API. RPC does not keep a copy of
291      *            the pAuthInfo pointer, and the client cannot retrieve it later
292      *            in the CoQueryProxyBlanket method. If this parameter is NULL,
293      *            DCOM uses the current proxy identity (which is either the
294      *            process token or the impersonation token). If the handle
295      *            refers to a structure, that identity is used. For Schannel,
296      *            this parameter must be either a pointer to a CERT_CONTEXT
297      *            structure that contains the client's X.509 certificate or is
298      *            NULL if the client wishes to make an anonymous connection to
299      *            the server. If a certificate is specified, the caller must not
300      *            free it as long as any proxy to the object exists in the
301      *            current apartment. For Snego, this member is either NULL,
302      *            points to a SEC_WINNT_AUTH_IDENTITY structure, or points to a
303      *            SEC_WINNT_AUTH_IDENTITY_EX structure. If it is NULL, Snego
304      *            will pick a list of authentication services based on those
305      *            available on the client computer. If it points to a
306      *            SEC_WINNT_AUTH_IDENTITY_EX structure, the structure's
307      *            PackageList member must point to a string containing a
308      *            comma-separated list of authentication service names and the
309      *            PackageListLength member must give the number of bytes in the
310      *            PackageList string. If PackageList is NULL, all calls using
311      *            Snego will fail. If COLE_DEFAULT_AUTHINFO is specified for
312      *            this parameter, DCOM will pick the authentication information
313      *            following its normal security blanket negotiation algorithm.
314      *            CoSetProxyBlanket will fail if pAuthInfo is set and one of the
315      *            cloaking flags is set in the dwCapabilities parameter.
316      * @param dwCapabilities
317      *            [in] The capabilities of this proxy. For a list of possible
318      *            values, see the EOLE_AUTHENTICATION_CAPABILITIES enumeration.
319      *            The only flags that can be set through this function are
320      *            EOAC_MUTUAL_AUTH, EOAC_STATIC_CLOAKING, EOAC_DYNAMIC_CLOAKING,
321      *            EOAC_ANY_AUTHORITY (this flag is deprecated),
322      *            EOAC_MAKE_FULLSIC, and EOAC_DEFAULT. Either
323      *            EOAC_STATIC_CLOAKING or EOAC_DYNAMIC_CLOAKING can be set if
324      *            pAuthInfo is not set and Schannel is not the authentication
325      *            service. (See Cloaking for more information.) If any
326      *            capability flags other than those mentioned here are set,
327      *            CoSetProxyBlanket will fail.
328      * @return This function can return the following values.
329      *
330      *         S_OK The function was successful.
331      *
332      *         E_INVALIDARG One or more arguments is invalid.
333      */
334     HRESULT CoSetProxyBlanket(Unknown pProxy, //
335             int dwAuthnSvc, //
336             int dwAuthzSvc, //
337             LPOLESTR pServerPrincName, //
338             int dwAuthnLevel, //
339             int dwImpLevel, //
340             Pointer pAuthInfo, // RPC_AUTH_IDENTITY_HANDLE
341             int dwCapabilities//

430     /**
431      * Allocates a block of task memory in the same way that IMalloc::Alloc
432      * does. CoTaskMemAlloc uses the default allocator to allocate a memory
433      * block in the same way that IMalloc::Alloc does. It is not necessary to
434      * call the CoGetMalloc function before calling CoTaskMemAlloc.
435      * <p>
436      * The initial contents of the returned memory block are undefined - there
437      * is no guarantee that the block has been initialized. The allocated block
438      * may be larger than cb bytes because of the space required for alignment
439      * and for maintenance information.
440      * </p>
441      * If cb is 0, CoTaskMemAlloc allocates a zero-length item and returns a
442      * valid pointer to that item. If there is insufficient memory available,
443      * CoTaskMemAlloc returns NULL. Applications should always check the return
444      * value from this function, even when requesting small amounts of memory,
445      * because there is no guarantee that the memory will be allocated.
446      * 
447      * @param cb
448      *            The size of the memory block to be allocated, in bytes.
449      * @return If the function succeeds, it returns the allocated memory block.
450      *         Otherwise, it returns NULL.
451      */
452     Pointer CoTaskMemAlloc(long cb);

454     /**
455      * Changes the size of a previously allocated block of task memory. This
456      * function changes the size of a previously allocated memory block in the
457      * same way that IMalloc::Realloc does. It is not necessary to call the
458      * CoGetMalloc function to get a pointer to the OLE allocator before calling
459      * CoTaskMemRealloc.
460      * <p>
461      * The pv parameter points to the beginning of the memory block. If pv is
462      * NULL, CoTaskMemRealloc allocates a new memory block in the same way as
463      * the CoTaskMemAlloc function. If pv is not NULL, it should be a pointer
464      * returned by a prior call to CoTaskMemAlloc.
465      * </p>
466      * <p>
467      * The cb parameter specifies the size of the new block. The contents of the
468      * block are unchanged up to the shorter of the new and old sizes, although
469      * the new block can be in a different location. Because the new block can
470      * be in a different memory location, the pointer returned by
471      * CoTaskMemRealloc is not guaranteed to be the pointer passed through the
472      * pv argument. If pv is not NULL and cb is 0, then the memory pointed to by
473      * pv is freed.
474      * </p>
475      * CoTaskMemRealloc returns a void pointer to the reallocated (and possibly
476      * moved) memory block. The return value is NULL if the size is 0 and the
477      * buffer argument is not NULL, or if there is not enough memory available
478      * to expand the block to the specified size. In the first case, the
479      * original block is freed; in the second case, the original block is
480      * unchanged. The storage space pointed to by the return value is guaranteed
481      * to be suitably aligned for storage of any type of object. To get a
482      * pointer to a type other than void, use a type cast on the return value.
483      * 
484      * @param pv
485      *            A pointer to the memory block to be reallocated. This
486      *            parameter can be NULL.
487      * @param cb
488      *            The size of the memory block to be reallocated, in bytes. This
489      *            parameter can be 0.
490      * @return If the function succeeds, it returns the reallocated memory
491      *         block. Otherwise, it returns NULL.
492      */
493     Pointer CoTaskMemRealloc(Pointer pv, long cb);

640     /**
641      * Carries out the clipboard shutdown sequence. It also releases the
642      * IDataObject pointer that was placed on the clipboard by the
643      * OleSetClipboard function.
644      * 
645      * @return {@link WinError#S_OK S_OK} on success.
646      *         <p>
647      *         {@link WinError#CLIPBRD_E_CANT_OPEN CLIPBRD_E_CANT_OPEN} The
648      *         Windows OpenClipboard function used within OleFlushClipboard
649      *         failed.
650      *         <p>
651      *         {@link WinError#CLIPBRD_E_CANT_CLOSE CLIPBRD_E_CANT_CLOSE} The
652      *         Windows CloseClipboard function used within OleFlushClipboard
653      *         failed.
654      *         <p>
655      *         <b>Remarks</b>
656      *         <p>
657      *         OleFlushClipboard renders the data from a data object onto the
658      *         clipboard and releases the IDataObject pointer to the data
659      *         object. While the application that put the data object on the
660      *         clipboard is running, the clipboard holds only a pointer to the
661      *         data object, thus saving memory. If you are writing an
662      *         application that acts as the source of a clipboard operation, you
663      *         can call the OleFlushClipboard function when your application is
664      *         closed, such as when the user exits from your application.
665      *         Calling OleFlushClipboard enables pasting and paste-linking of
666      *         OLE objects after application shutdown. Before calling
667      *         OleFlushClipboard, you can easily determine if your data is still
668      *         on the clipboard with a call to the OleIsCurrentClipboard
669      *         function.
670      *
671      *         OleFlushClipboard leaves all formats offered by the data transfer
672      *         object, including the OLE 1 compatibility formats, on the
673      *         clipboard so they are available after application shutdown. In
674      *         addition to OLE 1 compatibility formats, these include all
675      *         formats offered on a global handle medium (all except for
676      *         TYMED_FILE) and formatted with a null target device. For example,
677      *         if a data-source application offers a particular clipboard format
678      *         (say cfFOO) on an IStorage object, and calls the
679      *         OleFlushClipboard function, the storage object is copied into
680      *         memory and the hglobal memory handle is put on the clipboard.
681      *
682      *         To retrieve the information on the clipboard, you can call the
683      *         OleGetClipboard function from another application, which creates
684      *         a default data object, and the hglobal from the clipboard again
685      *         becomes a storage object. Furthermore, the FORMATETC enumerator
686      *         and the IDataObject::QueryGetData method would all correctly
687      *         indicate that the original clipboard format (cfFOO) is again
688      *         available on a TYMED_ISTORAGE.
689      *
690      *         To empty the clipboard, call the OleSetClipboard function
691      *         specifying a null value for its parameter. The application should
692      *         call this when it closes if there is no need to leave data on the
693      *         clipboard after shutdown, or if data will be placed on the
694      *         clipboard using the standard Windows clipboard functions.
695      */
696     HRESULT OleFlushClipboard();
